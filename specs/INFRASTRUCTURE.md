# Инфраструктура etraproject.ru на Timeweb Cloud

## Обзор

Архитектура рассчитана на высокие нагрузки, быстрый старт и минимальные затраты на DevOps.
Payload CMS 3.x встроен в Next.js как единое приложение — один контейнер обслуживает и фронт, и админку, и API.

---

## Что заказываем в Timeweb Cloud

### 1. App Platform (основной деплой)

**Назначение:** автодеплой Next.js + Payload из GitHub

| Параметр | Значение |
|----------|---------|
| Источник | GitHub репозиторий `site-b2c` |
| Runtime | Docker (Dockerfile) |
| Тариф | от 2-4 vCPU / 2-4 GB RAM |
| Автодеплой | push в `main` → build → deploy |
| Rollback | встроенный |

**Почему App Platform, а не VDS:**
- Zero-downtime деплой из коробки
- Встроенное логирование и мониторинг
- Автоматический HTTPS
- Горизонтальное масштабирование при росте нагрузки
- Не нужно настраивать nginx, certbot, systemd

**Альтернатива (если App Platform не подойдёт):**
Облачный сервер (VDS) 4 vCPU / 8 GB RAM / 80 GB NVMe + Docker Compose + Caddy

### 2. База данных (DBaaS PostgreSQL)

**Назначение:** основная БД для Payload CMS (контент, пользователи, заказы, каталог)

| Параметр | Значение |
|----------|---------|
| Движок | PostgreSQL 16 |
| Тариф | 1 vCPU / 2 GB RAM / 32 GB SSD (от 590 ₽/мес) |
| Локация | Москва или Санкт-Петербург |
| Бэкапы | автоматические (бесплатно) |
| Replica | по мере роста (read replica) |

**Почему DBaaS, а не PostgreSQL на VDS:**
- Автоматические бэкапы
- Мониторинг и алерты
- Масштабирование без даунтайма
- Отдельный сервис = безопасность (изоляция)

### 3. Хранилище S3

**Назначение:** медиафайлы (изображения товаров, баннеры, документы, видео)

| Параметр | Значение |
|----------|---------|
| Бакет | `etra-media` |
| Политика | публичное чтение для медиа, приватное для документов |
| Интеграция | `@payloadcms/storage-s3` |
| Ориентировочный объём | 10-50 GB на старте |
| S3 URL | https://s3.twcstorage.ru |
| Стоимость | 79 ₽/мес (10 ГБ) |

**Возможности:**
- S3-совместимый API (работает с AWS SDK)
- CDN-интеграция для ускорения отдачи статики
- Тройная репликация данных

### 4. Домен

**Назначение:** привязка домена `etraproject.ru` к App Platform

| Параметр | Значение |
|----------|---------|
| Домен | etraproject.ru |
| DNS | Timeweb Cloud DNS (или текущий регистратор) |
| SSL | автоматический Let's Encrypt |
| Записи | A → App Platform IP, CNAME www → @ |

### 5. Корпоративная почта (опционально)

**Назначение:** отправка transactional email (заказы, регистрация)

| Параметр | Значение |
|----------|---------|
| Домен | etraproject.ru |
| Ящики | noreply@etraproject.ru, support@etraproject.ru |
| Использование | SMTP для Payload email adapter |

**Альтернатива:** Resend.com (600 email/день бесплатно)

## Что НЕ нужно на старте

| Сервис Timeweb | Зачем | Когда понадобится |
|---------------|-------|------------------|
| Балансировщик | L4/L7 балансировка | При 2+ инстансах приложения |
| Kubernetes | Оркестрация контейнеров | При микросервисной архитектуре |
| Выделенный сервер | Dedicated CPU | При >50k DAU |
| Роутер | Приватная сеть | При нескольких VDS |
| Реестр контейнеров | Docker Registry | Если App Platform не подойдёт |
| AI-агенты | RAG / чат-боты | Отдельная задача |

## Ориентировочная стоимость (мес.)

| Сервис | Тариф | Стоимость |
|--------|-------|-----------|
| App Platform | 2 vCPU / 2 GB | ~1 200 ₽ |
| DBaaS PostgreSQL | 1 vCPU / 2 GB / 32 GB | ~590 ₽ |
| S3 (20 GB) | 20 GB хранение + трафик | ~100 ₽ |
| Домен (если через Timeweb) | .ru | ~200 ₽/год |
| Корп. почта | 2 ящика | ~100 ₽ |
| **Итого** | | **~2 000 – 2 500 ₽/мес** |

При росте нагрузки: увеличиваем App Platform тариф или добавляем инстансы + read replica PostgreSQL.

## Альтернативный вариант (VDS)

Если App Platform не покроет потребности (нужен больше контроль):

| Сервис | Конфигурация | Стоимость |
|--------|-------------|-----------|
| VDS (Cloud-100) | 4 vCPU / 8 GB / 80 GB NVMe | ~2 745 ₽ |
| DBaaS PostgreSQL | 1 vCPU / 2 GB / 32 GB | ~590 ₽ |
| S3 (20 GB) | 20 GB | ~100 ₽ |
| **Итого** | | **~3 400 ₽/мес** |

На VDS ставим: Docker Compose (Next.js + Caddy reverse proxy), автоматический HTTPS, CI/CD через GitHub Actions.

## Порядок создания

1. **S3 бакет** `etra-media` → получить access key / secret key
2. **DBaaS PostgreSQL** → получить connection string
3. **App Platform** → подключить GitHub repo, задать env-переменные
4. **Домен** → привязать DNS к App Platform
5. **Корп. почта** → настроить SPF/DKIM/DMARC
